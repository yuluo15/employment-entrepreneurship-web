# 学生端通知公告列表和详情 - 实现总结

## 功能概述

为学生端首页的"通知公告"模块添加完整的列表页和详情页，让学生可以查看所有历史通知和完整内容。

---

## 实现的功能

### 1. 通知公告列表页 (`/student/notices`)

**页面路径**: `src/views/mobile/notice/list.vue`

**功能特性**:
- 顶部导航栏（带返回按钮）
- 筛选功能（通知类型、发布者类型）
- 下拉刷新
- 上拉加载更多（分页）
- 通知卡片展示（类型标签、标题、发布者、时间、浏览量）
- 置顶通知标识（火焰图标）
- 点击跳转详情页
- 空状态提示

**筛选选项**:
- 通知类型：全部、活动通知、赛事通知、讲座通知、政策通知、一般通知
- 发布者：全部、省教育厅、本校通知

**时间显示优化**:
- 1小时内：显示"X分钟前"
- 24小时内：显示"X小时前"
- 7天内：显示"X天前"
- 超过7天：显示日期（YYYY-MM-DD）

### 2. 通知公告详情页 (`/student/notice/:id`)

**页面路径**: `src/views/mobile/notice/detail.vue`

**功能特性**:
- 顶部导航栏（带返回按钮）
- 通知类型标签
- 置顶标识
- 完整标题
- 发布者、发布时间、浏览量
- 富文本内容展示（支持HTML）
- 附件列表（支持下载）
- 加载状态
- 空状态提示

**内容样式优化**:
- 图片自适应宽度
- 段落间距优化
- 标题加粗
- 列表缩进
- 链接高亮

## 路由配置

在 `src/router/menu.ts` 的 `studentMenu` 中添加：

```typescript
// 通知公告列表
{
  path: 'notices',
  name: 'NoticeList',
  meta: { title: '通知公告', hideTabBar: true },
  component: () => import('@/views/mobile/notice/list.vue')
},
// 通知公告详情
{
  path: 'notice/:id',
  name: 'NoticeDetail',
  meta: { title: '通知详情', hideTabBar: true },
  component: () => import('@/views/mobile/notice/detail.vue')
}
```

**说明**:
- `hideTabBar: true` - 隐藏底部导航栏，提供更好的阅读体验
- 详情页使用动态路由参数 `:id`

## API接口

### 1. 获取通知列表

**接口**: `GET /api/mobile/notices`

**参数**:
```typescript
{
  pageNum: number      // 页码
  pageSize: number     // 每页数量
  noticeType?: string  // 通知类型（可选）
  publisherType?: string // 发布者类型（可选）
}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 50,
    "data": [
      {
        "noticeId": "1",
        "noticeTitle": "通知标题",
        "noticeType": "ACTIVITY",
        "noticeTypeText": "活动通知",
        "publishTime": "2024-01-15 10:30:00",
        "publisherType": "ADMIN",
        "publisherName": "省教育厅",
        "isTop": 1,
        "viewCount": 1250
      }
    ]
  }
}
```

### 2. 获取通知详情

**接口**: `GET /api/mobile/notice/:noticeId`

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "noticeId": "1",
    "noticeTitle": "通知标题",
    "noticeType": "ACTIVITY",
    "noticeTypeText": "活动通知",
    "noticeContent": "<p>通知内容HTML</p>",
    "publishTime": "2024-01-15 10:30:00",
    "publisherType": "ADMIN",
    "publisherName": "省教育厅",
    "isTop": 1,
    "viewCount": 1251,
    "attachments": [
      {
        "name": "文件名.pdf",
        "url": "https://example.com/file.pdf",
        "size": "2.5MB"
      }
    ]
  }
}
```

## 页面跳转流程

```
首页 (/student/home)
  ├─ 点击"更多" → 通知列表 (/student/notices)
  │                  ├─ 筛选通知类型/发布者
  │                  ├─ 下拉刷新
  │                  ├─ 上拉加载更多
  │                  └─ 点击通知 → 通知详情 (/student/notice/:id)
  │                                  ├─ 查看完整内容
  │                                  ├─ 下载附件
  │                                  └─ 返回列表
  └─ 点击单个通知 → 通知详情 (/student/notice/:id)
```

## 通知类型配置

| 类型代码 | 类型名称 | 标签颜色 | 说明 |
|---------|---------|---------|------|
| ACTIVITY | 活动通知 | primary（蓝色） | 校园活动、双选会等 |
| COMPETITION | 赛事通知 | warning（橙色） | 创业大赛、技能竞赛等 |
| LECTURE | 讲座通知 | success（绿色） | 就业指导讲座等 |
| POLICY | 政策通知 | danger（红色） | 就业政策、补贴政策等 |
| GENERAL | 一般通知 | default（灰色） | 其他通知 |

## 技术实现要点

### 1. 下拉刷新 + 上拉加载

```vue
<van-pull-refresh v-model="refreshing" @refresh="onRefresh">
  <van-list
    v-model:loading="loading"
    :finished="finished"
    finished-text="没有更多了"
    @load="onLoad"
  >
    <!-- 列表内容 -->
  </van-list>
</van-pull-refresh>
```

### 2. 筛选功能

```vue
<van-dropdown-menu>
  <van-dropdown-item 
    v-model="filterType" 
    :options="typeOptions" 
    @change="onFilterChange" 
  />
  <van-dropdown-item 
    v-model="filterPublisher" 
    :options="publisherOptions" 
    @change="onFilterChange" 
  />
</van-dropdown-menu>
```

### 3. 富文本内容展示

```vue
<div 
  class="notice-content"
  v-html="notice.noticeContent"
></div>
```

**安全提示**: 
- 后端需要对HTML内容进行XSS过滤
- 前端使用 `v-html` 时要确保内容来源可信

### 4. 时间格式化

```typescript
const formatTime = (time: string) => {
  const date = new Date(time)
  const now = new Date()
  const diff = now.getTime() - date.getTime()
  
  if (diff < 3600000) return Math.floor(diff / 60000) + '分钟前'
  if (diff < 86400000) return Math.floor(diff / 3600000) + '小时前'
  if (diff < 604800000) return Math.floor(diff / 86400000) + '天前'
  return time.substring(0, 10)
}
```

## 后端需要实现的功能

### 1. 通知列表接口

- 支持分页（pageNum, pageSize）
- 支持筛选（noticeType, publisherType）
- 只返回已发布的通知（status=1）
- 只返回学生可见的通知（权限控制）
- 置顶通知优先显示
- 按发布时间倒序排列

### 2. 通知详情接口

- 返回完整的通知内容（HTML格式）
- 返回附件列表
- 浏览次数+1
- 权限验证（学生是否有权查看）

### 3. 权限控制

学生可见的通知范围：
- 省级通知：`publisher_type='ADMIN'` 且 `target_audience='ALL'` 或 `'STUDENT'`
- 本校通知：`publisher_type='SCHOOL'` 且 `publisher_id=学生所在学校ID`
- 状态：只显示已发布的通知（`status=1`）

  </div>

```typescript
// 跳转通知列表
const toNoticeList = () => {
  router.push('/student/notices')

const toNoticeDetail = (noticeId: string) => {
  router.push(`/student/notice/${noticeId}`)
}
```

## 总结

### 实现的价值

1. **完善功能闭环**: 首页显示最新通知 → 列表查看所有通知 → 详情查看完整内容
2. **提升用户体验**: 支持筛选、刷新、分页，方便学生查找历史通知
3. **信息传达完整**: 支持富文本内容和附件下载，确保信息完整传达
4. **实现成本低**: 复用已有API，只需创建2个页面，30分钟完成

### 功能特色

- 下拉刷新 + 上拉加载（流畅的移动端体验）
- 筛选功能（快速找到需要的通知）
- 时间优化显示（相对时间更友好）
- 富文本内容（支持图片、链接、格式）
- 附件下载（完整的信息传达）
- 置顶标识（重要通知突出显示）

### 后续优化建议

1. **搜索功能**: 添加通知标题搜索
2. **收藏功能**: 收藏重要通知方便查看
3. **已读标识**: 标记已读/未读通知
4. **推送通知**: 新通知发布时推送给学生
5. **分享功能**: 分享通知给其他同学
