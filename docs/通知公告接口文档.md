# 通知公告接口文档

## 接口概述

通知公告模块提供公告的增删改查、发布/停用、置顶等功能。

---

## 1. 获取公告列表（分页）

### 接口信息
- **接口路径**: `GET /api/notice/list`
- **接口描述**: 分页查询公告列表，支持按标题、类型、状态筛选
- **请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| noticeTitle | string | 否 | 标题（模糊查询） |
| noticeType | string | 否 | 类型：1=通知, 2=公告, 3=政策, 4=新闻 |
| status | number | 否 | 状态：0=草稿, 1=已发布 |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "noticeId": "1",
        "noticeTitle": "关于2026届毕业生就业工作的通知",
        "noticeType": "1",
        "noticeContent": "各高校...",
        "status": 1,
        "isTop": 1,
        "viewCount": 1523,
        "createBy": "admin",
        "createTime": "2026-02-01 10:00:00",
        "updateTime": "2026-02-01 10:00:00",
        "publishTime": "2026-02-01 10:30:00"
      }
    ],
    "total": 50
  }
}
```

### SQL 实现示例

```sql
-- 基础查询
SELECT 
  notice_id AS "noticeId",
  notice_title AS "noticeTitle",
  notice_type AS "noticeType",
  notice_content AS "noticeContent",
  status,
  is_top AS "isTop",
  view_count AS "viewCount",
  attachments,
  create_by AS "createBy",
  create_time AS "createTime",
  update_time AS "updateTime",
  publish_time AS "publishTime"
FROM sys_notice
WHERE 1=1
  AND (#{noticeTitle} IS NULL OR notice_title LIKE CONCAT('%', #{noticeTitle}, '%'))
  AND (#{noticeType} IS NULL OR notice_type = #{noticeType})
  AND (#{status} IS NULL OR status = #{status})
ORDER BY 
  is_top DESC,           -- 置顶的排在前面
  publish_time DESC,     -- 按发布时间降序
  create_time DESC       -- 未发布的按创建时间降序
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};

-- 总数查询
SELECT COUNT(*) 
FROM sys_notice
WHERE 1=1
  AND (#{noticeTitle} IS NULL OR notice_title LIKE CONCAT('%', #{noticeTitle}, '%'))
  AND (#{noticeType} IS NULL OR notice_type = #{noticeType})
  AND (#{status} IS NULL OR status = #{status});
```

## 2. 新增公告

### 接口信息
- **接口路径**: `POST /api/notice/add`
- **接口描述**: 新增公告（可保存为草稿或直接发布）
- **请求体**:

```json
{
  "noticeTitle": "关于2026届毕业生就业工作的通知",
  "noticeType": "1",
  "noticeContent": "各高校...",
  "isTop": 0,
  "status": 0
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| noticeTitle | string | 是 | 标题，最大200字符 |
| noticeType | string | 是 | 类型：1=通知, 2=公告, 3=政策, 4=新闻 |
| noticeContent | string | 是 | 内容 |
| isTop | number | 否 | 是否置顶：0=否, 1=是，默认0 |
| status | number | 否 | 状态：0=草稿, 1=已发布，默认0 |

### 响应数据

```json
{
  "code": 200,
  "msg": "新增成功",
  "data": null
}
```

### SQL 实现示例

```sql
INSERT INTO sys_notice (
  notice_id,
  notice_title,
  notice_type,
  notice_content,
  status,
  is_top,
  view_count,
  create_by,
  create_time,
  update_time,
  publish_time
) VALUES (
  #{noticeId},           -- 使用UUID或雪花算法生成
  #{noticeTitle},
  #{noticeType},
  #{noticeContent},
  #{status},
  #{isTop},
  0,                     -- 初始阅读量为0
  #{createBy},           -- 从登录用户获取
  CURRENT_TIMESTAMP,
  CURRENT_TIMESTAMP,
  CASE WHEN #{status} = 1 THEN CURRENT_TIMESTAMP ELSE NULL END  -- 如果是发布状态，设置发布时间
);
```

## 3. 更新公告

### 接口信息
- **接口路径**: `PUT /api/notice/update`
- **接口描述**: 更新公告信息
- **请求体**:

```json
{
  "noticeId": "1",
  "noticeTitle": "关于2026届毕业生就业工作的通知（修改）",
  "noticeType": "1",
  "noticeContent": "各高校...",
  "isTop": 1,
  "status": 1
}
```

### 响应数据

```json
{
  "code": 200,
  "msg": "更新成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE sys_notice
SET 
  notice_title = #{noticeTitle},
  notice_type = #{noticeType},
  notice_content = #{noticeContent},
  is_top = #{isTop},
  status = #{status},
  update_time = CURRENT_TIMESTAMP,
  -- 如果从草稿改为发布，更新发布时间
  publish_time = CASE 
    WHEN #{status} = 1 AND publish_time IS NULL 
    THEN CURRENT_TIMESTAMP 
    ELSE publish_time 
  END
WHERE notice_id = #{noticeId};
```

## 5. 发布公告

### 接口信息
- **接口路径**: `PUT /api/notice/publish/{noticeId}`
- **接口描述**: 将草稿状态的公告发布
- **路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| noticeId | string | 是 | 公告ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "发布成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE sys_notice
SET 
  status = 1,
  publish_time = CURRENT_TIMESTAMP,
  update_time = CURRENT_TIMESTAMP
WHERE notice_id = #{noticeId} AND status = 0;
```

## 6. 停用公告

### 接口信息
- **接口路径**: `PUT /api/notice/unpublish/{noticeId}`
- **接口描述**: 将已发布的公告停用（改为草稿状态）
- **路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| noticeId | string | 是 | 公告ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "停用成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE sys_notice
SET 
  status = 0,
  update_time = CURRENT_TIMESTAMP
WHERE notice_id = #{noticeId} AND status = 1;
```

## 业务逻辑说明

### 1. 公告状态流转

```
草稿(0) ──发布──> 已发布(1)
   ↑                 │
   └─────停用────────┘
```

- 新增时可以直接保存为草稿或发布
- 草稿可以发布，已发布可以停用
- 删除操作不受状态限制

### 2. 置顶规则

- 置顶的公告在列表中始终排在最前面
- 多个置顶公告按发布时间降序排列
- 置顶状态不受发布/停用影响

### 3. 发布时间规则

- 新增时如果 `status=1`，则 `publish_time` 设置为当前时间
- 新增时如果 `status=0`，则 `publish_time` 为 NULL
- 草稿发布时，设置 `publish_time` 为当前时间
- 停用后再次发布，`publish_time` 不变（保留首次发布时间）

### 4. 阅读量统计

- 新增时 `view_count` 初始化为 0
- 前端展示公告详情时，调用单独的接口增加阅读量（可选功能）
- 阅读量只增不减

### 5. 附件处理

- 附件功能已移除，不再支持附件上传

---

## 排序规则

列表查询时的排序优先级：

1. **置顶优先**: `is_top DESC` - 置顶的排在前面
2. **发布时间**: `publish_time DESC` - 已发布的按发布时间降序
3. **创建时间**: `create_time DESC` - 草稿按创建时间降序

## 前端调用示例

```typescript
import { getNoticeList, addNotice, publishNotice } from '@/api/notice'

// 1. 查询列表
const loadList = async () => {
  const res = await getNoticeList({
    pageNum: 1,
    pageSize: 10,
    noticeTitle: '就业',
    status: 1
  })
  console.log(res.data.list)
}

// 2. 新增公告（保存为草稿）
const addDraft = async () => {
  await addNotice({
    noticeTitle: '测试公告',
    noticeType: '1',
    noticeContent: '内容',
    isTop: 0,
    status: 0  // 草稿
  })
}

// 3. 发布公告
const publish = async (noticeId: string) => {
  await publishNotice(noticeId)
}
```

## 注意事项

1. **内容格式**: `noticeContent` 字段存储纯文本内容
2. **权限控制**: 所有接口都需要管理员权限，建议在后端统一拦截
3. **软删除**: 如果需要保留删除记录，可以添加 `del_flag` 字段实现软删除
4. **阅读量**: 当前设计中阅读量只在新增时初始化，如需统计真实阅读量，需要额外实现详情查看接口并增加计数
5. **响应格式**: 列表接口返回的数据在 `data.data` 中，不是 `data.list`
