# 字典数据补充说明

## 📋 概述

本文档说明了系统中缺失的字典数据，这些字典在各个接口中被使用但在数据库中缺失。

---

## 🔍 分析过程

通过分析以下接口文档，发现了需要补充的字典项：

1. 企业端简历管理接口文档
2. 企业端人才库接口文档
3. 学生端项目申请接口文档
4. 通知公告接口文档
5. 岗位审核接口文档
6. 职位管理接口文档

---

## 📊 需要补充的字典类型

### 1. 投递状态 (sys_delivery_status)
**用途**: 企业端简历管理、人才库管理

| 值 | 标签 | 说明 |
|---|---|---|
| DELIVERED | 待处理 | 简历已投递，待HR处理 |
| INTERVIEW | 面试中 | 已安排面试 |
| OFFER | 已录用 | 已发放Offer |
| REJECTED | 已拒绝 | 简历不合适 |

**使用位置**:
- `biz_job_delivery.status` 字段

## 📝 执行步骤

### 1. 备份数据库
```bash
pg_dump -U your_username -d your_database > backup_$(date +%Y%m%d).sql
```

### 2. 执行补充SQL
```bash
psql -U your_username -d your_database -f docs/db/sys_dict_补充数据.sql
```

### 3. 验证数据
```sql
-- 查看新增的字典类型
SELECT dict_type, dict_name, remark FROM sys_dict_type 
WHERE create_time >= CURRENT_DATE 
ORDER BY create_time DESC;

-- 查看新增的字典数据
SELECT dict_type, dict_label, dict_value FROM sys_dict_data 
WHERE create_time >= CURRENT_DATE 
ORDER BY dict_type, dict_sort;
```

## ⚠️ 注意事项

1. **ID冲突**: 如果执行时提示ID冲突，请修改SQL中的UUID
2. **数据验证**: 执行后务必验证数据是否正确插入
3. **业务影响**: 补充字典后，相关接口才能正常工作
4. **前端适配**: 前端下拉框需要调用字典接口获取选项
