# 管理员首页接口文档

## 接口概述

管理员首页需要一个统一的概览接口，返回平台核心指标、待办事项、学校排行、创业领域分布和最新入驻企业等数据。

---

## 1. 获取首页概览数据

### 接口信息
- **接口路径**: `GET /api/home/overview`
- **接口描述**: 获取管理员首页所有数据（KPI指标、待办审批、学校排行、创业领域分布、最新企业）
- **请求参数**: 无

### 字段说明

#### 核心指标
| 字段名 | 类型 | 说明 |
|--------|------|------|
| schoolCount | number | 入驻学校总数 |
| companyCount | number | 入驻企业总数 |
| studentCount | number | 学生总数 |
| teacherCount | number | 教师总数 |
| pendingJobs | number | 待审核岗位数量 |
| pendingProjects | number | 待审核创业项目数量 |

#### 学校排行 (schoolRank)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| schoolId | string | 学校ID |
| schoolName | string | 学校名称 |
| studentCount | number | 该校学生数量 |

**排序规则**: 按学生数量降序排列，取前10所学校

#### 创业领域分布 (domainDistribution)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| domain | string | 领域字典值 |
| domainLabel | string | 领域显示名称 |
| count | number | 该领域项目数量 |

**统计规则**: 统计所有状态的创业项目（包括审核中、孵化中、已驳回、已完结）

#### 最新入驻企业 (latestCompanies)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| companyId | string | 企业ID |
| companyName | string | 企业名称 |
| industry | string | 所属行业 |
| jobCount | number | 该企业发布的岗位数量 |
| createTime | string | 入驻时间 |

**排序规则**: 按入驻时间降序排列，取最新5家企业

## 后端实现参考

### SQL 示例

```sql
-- 1. 核心指标统计
SELECT 
  (SELECT COUNT(*) FROM sys_school) AS schoolCount,
  (SELECT COUNT(*) FROM sys_company) AS companyCount,
  (SELECT COUNT(*) FROM biz_student) AS studentCount,
  (SELECT COUNT(*) FROM biz_teacher) AS teacherCount,
  (SELECT COUNT(*) FROM biz_job WHERE audit = 0) AS pendingJobs,
  (SELECT COUNT(*) FROM biz_project WHERE status = 0) AS pendingProjects;

-- 2. 学校学生数量排行 (TOP 10)
SELECT 
  s.id AS schoolId,
  s.school_name AS schoolName,
  COUNT(st.id) AS studentCount
FROM sys_school s
LEFT JOIN biz_student st ON s.id = st.school_id
GROUP BY s.id, s.school_name
ORDER BY studentCount DESC
LIMIT 10;

-- 3. 创业项目领域分布
SELECT 
  p.domain,
  d.dict_label AS domainLabel,
  COUNT(*) AS count
FROM biz_project p
LEFT JOIN sys_dict_data d ON p.domain = d.dict_value AND d.dict_type = 'project_domain'
GROUP BY p.domain, d.dict_label
ORDER BY count DESC;

-- 4. 最新入驻企业 (TOP 5)
SELECT 
  c.id AS companyId,
  c.company_name AS companyName,
  c.industry,
  COUNT(j.id) AS jobCount,
  c.create_time AS createTime
FROM sys_company c
LEFT JOIN biz_job j ON c.id = j.company_id
GROUP BY c.id, c.company_name, c.industry, c.create_time
ORDER BY c.create_time DESC
LIMIT 5;
```

### 实现要点

1. **性能优化**: 
   - 使用子查询或多次查询分别获取各部分数据
   - 避免复杂的多表关联导致性能问题
   - 可以考虑缓存核心指标数据（如学校数、企业数等变化不频繁的数据）

2. **数据完整性**:
   - 学校排行中，如果学校没有学生，studentCount 应为 0
   - 企业列表中，如果企业没有发布岗位，jobCount 应为 0
   - 创业领域分布需要关联字典表获取 domainLabel

3. **待办统计**:
   - pendingJobs: 统计 `biz_job` 表中 `audit = 0` 的记录
   - pendingProjects: 统计 `biz_project` 表中 `status = 0` 的记录

## 前端调用示例

```typescript
import { getHomeOverview } from '@/api/home'

const loadData = async () => {
  try {
    const res = await getHomeOverview()
    const data = res.data
    
    // 使用返回的数据更新页面
    console.log('学校数量:', data.schoolCount)
    console.log('待审核岗位:', data.pendingJobs)
    console.log('学校排行:', data.schoolRank)
  } catch (error) {
    console.error('加载失败', error)
  }
}
```

## 注意事项

1. 这是一个**统一的概览接口**，一次请求返回所有首页需要的数据，避免多次请求
2. 数据量不大，不需要分页
3. 学校排行只取前10所，企业列表只取最新5家
4. 创业领域分布需要关联字典表获取中文标签
5. 所有统计数据都是实时计算的，如果数据量大可以考虑定时任务预计算
