# 个人中心接口文档

## 接口概述

个人中心模块提供用户基本信息查询、修改和密码修改功能。

---

## 1. 获取当前用户信息

### 接口信息
- **接口路径**: `GET /api/user/info`
- **接口描述**: 获取当前登录用户的详细信息
- **请求参数**: 无（从登录token中获取用户ID）

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": "1",
    "loginIdentity": "admin",
    "email": "admin@example.com",
    "phone": "13800138000",
    "nickname": "管理员",
    "realName": "张三",
    "avatar": "https://example.com/avatar.jpg",
    "gender": 1,
    "roleKey": "ROLE_ADMIN",
    "ownerId": null,
    "status": 1,
    "createTime": "2026-01-01 10:00:00",
    "loginDate": "2026-02-11 14:30:00"
  }
}
```

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 用户ID |
| loginIdentity | string | 登录账号 |
| email | string | 邮箱 |
| phone | string | 手机号 |
| nickname | string | 昵称 |
| realName | string | 真实姓名 |
| avatar | string | 头像URL |
| gender | number | 性别：0=未知, 1=男, 2=女 |
| roleKey | string | 角色标识：ROLE_ADMIN/ROLE_SCHOOL/ROLE_COMPANY/ROLE_TEACHER/ROLE_STUDENT |
| ownerId | string | 所属机构ID（学校ID或企业ID） |
| status | number | 账号状态：0=禁用, 1=正常 |
| createTime | string | 创建时间 |
| loginDate | string | 最近登录时间 |

### SQL 实现示例

```sql
SELECT 
  id,
  login_identity AS "loginIdentity",
  email,
  phone,
  nickname,
  real_name AS "realName",
  avatar,
  gender,
  role_key AS "roleKey",
  owner_id AS "ownerId",
  status,
  create_time AS "createTime",
  login_date AS "loginDate"
FROM sys_user
WHERE id = #{userId};  -- 从登录token中获取
```

## 2. 更新用户信息

### 接口信息
- **接口路径**: `PUT /api/user/update`
- **接口描述**: 更新当前用户的基本信息（不包括密码）
- **请求体**:

```json
{
  "id": "1",
  "email": "newemail@example.com",
  "phone": "13900139000",
  "nickname": "新昵称",
  "realName": "李四",
  "avatar": "https://example.com/new-avatar.jpg",
  "gender": 2
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 用户ID |
| email | string | 否 | 邮箱 |
| phone | string | 否 | 手机号 |
| nickname | string | 否 | 昵称，最大50字符 |
| realName | string | 否 | 真实姓名，最大50字符 |
| avatar | string | 否 | 头像URL |
| gender | number | 否 | 性别：0=未知, 1=男, 2=女 |

### 响应数据

```json
{
  "code": 200,
  "msg": "更新成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE sys_user
SET 
  email = #{email},
  phone = #{phone},
  nickname = #{nickname},
  real_name = #{realName},
  avatar = #{avatar},
  gender = #{gender},
  update_time = CURRENT_TIMESTAMP
WHERE id = #{id};
```

### 业务规则

1. **权限验证**: 用户只能修改自己的信息，需要验证 `id` 是否与登录用户一致
2. **字段验证**:
   - 邮箱格式验证
   - 手机号格式验证（11位数字，1开头）
   - 昵称和真实姓名长度限制
3. **不可修改字段**: `loginIdentity`、`roleKey`、`ownerId`、`status` 等字段不允许用户自行修改

## 3. 修改密码

### 接口信息
- **接口路径**: `POST /api/user/changePassword`
- **接口描述**: 修改当前用户的登录密码
- **请求体**:

```json
{
  "userId": "1",
  "oldPassword": "123456",
  "newPassword": "newpass123"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | string | 是 | 用户ID |
| oldPassword | string | 是 | 旧密码（明文） |
| newPassword | string | 是 | 新密码（明文），至少6位 |

### 响应数据

**成功**:
```json
{
  "code": 200,
  "msg": "密码修改成功",
  "data": null
}
```

**失败（旧密码错误）**:
```json
{
  "code": 400,
  "msg": "旧密码错误",
  "data": null
}
```

### SQL 实现示例

```sql
-- 1. 验证旧密码
SELECT id, password
FROM sys_user
WHERE id = #{userId};

-- 在Java代码中使用BCrypt验证旧密码
-- if (!BCrypt.checkpw(oldPassword, dbPassword)) {
--     throw new BusinessException("旧密码错误");
-- }

-- 2. 更新新密码
UPDATE sys_user
SET 
  password = #{newPasswordHash},  -- 使用BCrypt加密后的密码
  update_time = CURRENT_TIMESTAMP
WHERE id = #{userId};
```

### 业务规则

1. **权限验证**: 用户只能修改自己的密码，需要验证 `userId` 是否与登录用户一致
2. **旧密码验证**: 必须先验证旧密码是否正确
3. **密码加密**: 
   - 前端传输明文密码（建议使用HTTPS）
   - 后端使用BCrypt加密后存储
   - 不要使用MD5等弱加密算法
4. **密码强度**: 
   - 最少6位
   - 建议要求包含字母、数字、符号
5. **修改后处理**: 
   - 密码修改成功后，前端应该清除token并跳转到登录页
   - 可选：发送邮件或短信通知用户

## 注意事项

1. **密码安全**:
   - 使用HTTPS传输密码
   - 后端使用BCrypt加密存储
   - 不要在日志中记录密码
   - 密码修改后强制重新登录

2. **权限控制**:
   - 所有接口都需要登录
   - 用户只能操作自己的信息
   - 从token中获取用户ID，不要完全信任前端传来的ID

3. **数据验证**:
   - 邮箱格式验证
   - 手机号格式验证
   - 密码长度和强度验证
   - 防止SQL注入和XSS攻击

4. **错误处理**:
   - 旧密码错误时返回明确的错误信息
   - 邮箱或手机号重复时给出提示
   - 账号被禁用时拒绝操作

5. **用户体验**:
   - 修改密码后自动退出登录
   - 提供密码强度提示
   - 重要信息修改后发送通知

## 数据库表结构参考

根据提供的 `sys_user` 表结构：

```sql
-- 用户表字段
id              -- 用户ID
login_identity  -- 登录账号（不可修改）
email           -- 邮箱（可修改）
phone           -- 手机号（可修改）
password        -- 密码（BCrypt加密）
nickname        -- 昵称（可修改）
real_name       -- 真实姓名（可修改）
avatar          -- 头像URL（可修改）
gender          -- 性别（可修改）
role_key        -- 角色标识（不可修改）
owner_id        -- 所属机构ID（不可修改）
status          -- 账号状态（管理员修改）
is_deleted      -- 逻辑删除标记
login_ip        -- 最近登录IP
login_date      -- 最近登录时间
create_by       -- 创建者
create_time     -- 创建时间

```
