# 教师端框架设计说明

## 设计理念

教师端采用**移动端（Vant）**技术栈，与学生端保持一致的用户体验。主要功能是指导创业项目，支持随时随地查看项目并发表指导意见。

---

## 整体架构

### 技术栈
- **UI框架**: Vant 4.x（移动端组件库）
- **路由**: Vue Router 4.x
- **状态管理**: Pinia
- **布局**: 底部TabBar导航 + KeepAlive缓存

### 目录结构
```
src/
├── layout/
│   └── teacher/
│       └── index.vue          # 教师端布局（TabBar + KeepAlive）
├── views/
│   └── teacher/
│       ├── home/
│       │   └── index.vue      # 工作台
│       ├── projects/
│       │   └── index.vue      # 项目列表（全网/本校切换）
│       ├── guidance/
│       │   └── index.vue      # 指导记录
│       └── profile/
│           └── index.vue      # 个人中心
├── store/
│   └── teacherStore.ts        # 教师端状态管理
└── api/
    └── teacher.ts             # 教师端API（待创建）
```

## 核心功能模块

### 1. 工作台（/teacher/home）

**功能特点**:
- 教师个人信息展示（头像、姓名、职称、学院）
- KPI数据卡片（指导次数、指导项目、评分）
- 待办事项（待指导项目、本周新增项目）
- 待指导项目列表（最新5个）
- 快捷入口（全网项目、本校项目、指导记录、数据统计）
- 最近指导记录

**设计亮点**:
- 渐变色顶部卡片，视觉效果好
- 待办事项突出显示，引导教师及时指导
- 快捷入口方便快速访问常用功能

### 2. 项目列表（/teacher/projects）

**功能特点**:
- **Tab切换**: 全网项目 / 本校项目
- **搜索**: 支持项目名称、学生姓名搜索
- **筛选**: 按状态（待审核、孵化中、已驳回、已落地）、领域筛选
- **项目卡片**: 显示项目名称、学生信息、学校、学院、领域、团队规模、指导次数
- **状态标签**: 不同状态用不同颜色标签区分
- **下拉刷新**: 支持下拉刷新和上拉加载更多
- **滚动保持**: 从详情页返回时保持滚动位置

**全网项目 vs 本校项目**:
- **全网项目**: 显示所有学校的创业项目，教师可以跨校指导
- **本校项目**: 只显示本校学生的创业项目，重点关注

**设计亮点**:
- Tab切换清晰，满足不同场景需求
- 项目卡片信息丰富，一目了然
- 待指导项目用橙色标签突出显示

### 3. 指导记录（/teacher/guidance）

**功能特点**:
- 顶部统计（总指导次数、指导项目数、本月指导）
- 搜索项目名称
- 指导记录列表（按时间倒序）
- 每条记录包含：
  - 项目信息（名称、学生、领域）
  - 指导内容
  - 指导时间
- 点击项目可跳转到项目详情

**设计亮点**:
- 统计数据直观展示教师工作量
- 记录卡片设计清晰，区分项目信息和指导内容
- 支持快速回顾历史指导

### 4. 个人中心（/teacher/profile）

**功能特点**:
- 教师个人信息展示（头像、姓名、职称、学院）
- 统计卡片（指导次数、指导项目、评分）
- 个人信息详情（工号、学院、职称、专业领域、手机号、邮箱）
- 功能菜单：
  - 个人信息编辑
  - 修改密码
  - 通知设置
  - 关于我们
  - 帮助与反馈
- 退出登录

**设计亮点**:
- 渐变色顶部卡片与工作台保持一致
- 统计卡片悬浮在顶部，视觉效果好
- 功能菜单清晰，图标配色统一

## 组件复用策略

### 复用学生端组件
1. **项目详情页** (`/views/mobile/project/detail.vue`)
   - 教师端和学生端都需要查看项目详情
   - 教师端在详情页可以发表指导评论

2. **个人信息编辑** (`/views/mobile/profile/info.vue`)
   - 基本信息编辑逻辑相同

3. **修改密码** (`/views/mobile/settings/password.vue`)
   - 密码修改逻辑相同

### 教师端独有组件
1. **工作台** - 展示教师特有的KPI和待办
2. **项目列表** - 支持全网/本校切换，教师视角
3. **指导记录** - 教师专属功能

---

## 权限设计

### 数据权限
- **全网项目**: 教师可以查看所有学校的创业项目
- **本校项目**: 只显示本校学生的项目
- **指导权限**: 教师可以对任何项目发表指导意见
- **编辑权限**: 教师不能编辑项目内容，只能指导

### 接口权限
- 所有接口需要验证教师身份（role_key = 'teacher'）
- 指导记录只能查看自己的指导
- 统计数据只显示自己的数据

## 与学生端的区别

| 特性 | 学生端 | 教师端 |
|------|--------|--------|
| 主要功能 | 找工作、发布项目 | 指导创业项目 |
| TabBar | 首页、职位、创业、我的 | 工作台、项目、指导、我的 |
| 项目列表 | 浏览项目、收藏 | 全网/本校切换、指导 |
| 项目详情 | 查看、收藏 | 查看、发表指导 |
| 核心操作 | 投递简历、发布项目 | 发表指导评论 |

## 设计优势

1. **移动优先**: 符合教师随时随地指导的使用场景
2. **体验统一**: 与学生端保持一致的技术栈和设计风格
3. **功能聚焦**: 专注于指导功能，界面简洁清晰
4. **组件复用**: 复用学生端组件，减少开发成本
5. **全网视野**: 支持跨校指导，扩大教师影响力
6. **本校优先**: 本校项目单独Tab，方便重点关注

---

## 技术亮点

1. **KeepAlive缓存**: 工作台和项目列表页面缓存，提升体验
2. **滚动保持**: 从详情页返回时保持滚动位置
3. **下拉刷新**: 支持下拉刷新和上拉加载更多
4. **Tab切换**: 全网/本校项目无缝切换
5. **角标提示**: TabBar显示待指导项目数量
6. **渐变设计**: 顶部卡片使用渐变色，视觉效果好

---

## 总结

教师端采用移动端设计，与学生端形成完整的移动端生态。通过"全网项目"和"本校项目"的设计，既满足教师跨校指导的需求，又突出本校项目的重点关注。整体框架简洁清晰，功能聚焦，符合教师的实际使用场景。
