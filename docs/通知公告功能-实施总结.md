# 通知公告功能 - 实施总结

## 📋 问题回顾

### 原有问题

1. ❌ 学生端首页看不到管理员和学校发布的公告
2. ❌ 学校端发布公告后不能编辑和删除
3. ❌ 目标受众逻辑过于复杂（区分教师和学生）
4. ❌ 附件功能不需要但代码中存在
5. ❌ 教师端没有公告功能

---

## ✅ 解决方案

### 1. 数据库字段使用

| 字段 | 管理员发布 | 学校发布 | 说明 |
|------|-----------|----------|------|
| publisher_type | 'admin' | 'school' | 发布者类型 |
| publisher_id | NULL | 学校ID | 发布者ID |
| target_audience | 'all'/'student' | 'student' | 目标受众 |
| create_by | 用户ID | 用户ID | 创建人 |

### 2. 权限控制

| 角色 | 可查看 | 可新增 | 可编辑 | 可删除 | 可置顶 |
|------|--------|--------|--------|--------|--------|
| 管理员 | 全部 | ✅ | 全部 | 全部 | ✅ |
| 学校 | 管理员+本校 | ✅ | 本校 | 本校 | ❌ |
| 学生 | 管理员+本校 | ❌ | ❌ | ❌ | ❌ |

### 3. 简化逻辑

- **学校发布**: 不再区分教师和学生，本校都能看
- **目标受众**: 学校固定为'student'，管理员可选'all'或'student'
- **权限判断**: 通过 `publisher_type` 和 `publisher_id` 判断

### 2. 学校端接口

需要实现的接口：

#### GET /api/school/notice/list
```sql
WHERE (
  (n.publisher_type = 'admin' AND n.status = 1)
  OR (n.publisher_type = 'school' AND n.publisher_id = #{schoolId})
)
```

#### POST /api/school/notice/add
```sql
publisher_type = 'school'
publisher_id = #{schoolId}
target_audience = 'student'
```

#### PUT /api/school/notice/update
```sql
WHERE notice_id = #{noticeId}
  AND publisher_type = 'school'
  AND publisher_id = #{schoolId}
```

#### DELETE /api/school/notice/delete/{noticeId}
```sql
WHERE notice_id = #{noticeId}
  AND publisher_type = 'school'
  AND publisher_id = #{schoolId}
```
