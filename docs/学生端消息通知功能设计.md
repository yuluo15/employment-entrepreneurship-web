# 学生端消息通知功能 - 设计方案

## 📋 功能概述

学生端需要展示企业发送的面试通知和Offer通知，让学生能够及时了解求职进展。

## 🎯 设计方案

### 1. 展示位置

在"我的"页面添加"消息通知"入口，放在"我发布的项目"下方。

#### 入口设计

```
我的页面
├── 个人信息卡片
├── 数据统计（已投递、待面试、收藏、Offer）
├── 我的在线简历
├── 我发布的项目
├── 消息通知 ⭐ 新增
│   ├── 显示未读消息数量（红点角标）
│   └── 点击进入消息列表页
└── 账号与安全
```

### 3. 数据模型

#### sys_message 表结构

```sql
CREATE TABLE sys_message (
  id varchar(64) PRIMARY KEY,
  receiver_id varchar(64) NOT NULL,           -- 接收人ID（学生ID）
  title varchar(200) NOT NULL,                -- 标题
  content text,                               -- 内容
  type integer NOT NULL,                      -- 消息类型（1:系统通知, 2:面试通知, 3:offer通知）
  is_read integer DEFAULT 0,                  -- 阅读状态（0:未读, 1:已读）
  ref_id varchar(64),                         -- 关联业务ID（投递ID）
  create_time timestamp DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_message_receiver ON sys_message(receiver_id);
CREATE INDEX idx_message_type ON sys_message(type);
CREATE INDEX idx_message_read ON sys_message(is_read);
CREATE INDEX idx_message_time ON sys_message(create_time);
```

## 📝 接口设计

### 1. 获取消息列表

**接口**: `GET /api/mobile/messages`

**请求参数**:
```typescript
interface MessageQuery {
  pageNum: number
  pageSize: number
  type?: number  // 消息类型：1=系统通知, 2=面试通知, 3=Offer通知
  isRead?: number  // 阅读状态：0=未读, 1=已读
}
```

**响应数据**:
```typescript
interface MessageItem {
  id: string
  title: string
  content: string
  type: number
  isRead: number
  refId: string | null
  createTime: string
  // 扩展字段
  companyName?: string  // 公司名称（从关联的投递记录获取）
  positionName?: string  // 职位名称（从关联的投递记录获取）
}

interface MessageListResponse {
  total: number
  data: MessageItem[]
}
```

**SQL实现**:
```sql
SELECT 
  m.id,
  m.title,
  m.content,
  m.type,
  m.is_read AS "isRead",
  m.ref_id AS "refId",
  m.create_time AS "createTime",
  -- 关联查询公司和职位信息
  c.name AS "companyName",
  p.position_name AS "positionName"
FROM sys_message m
LEFT JOIN biz_job_application ja ON m.ref_id = ja.id
LEFT JOIN biz_position p ON ja.position_id = p.id
LEFT JOIN sys_company c ON p.company_id = c.id
WHERE m.receiver_id = #{studentId}
  AND (#{type} IS NULL OR m.type = #{type})
  AND (#{isRead} IS NULL OR m.is_read = #{isRead})
ORDER BY m.create_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize}
```

### 2. 获取消息详情

**接口**: `GET /api/mobile/messages/{id}`

**响应数据**:
```typescript
interface MessageDetail {
  id: string
  title: string
  content: string
  type: number
  isRead: number
  refId: string | null
  createTime: string
  // 扩展字段
  companyName?: string
  positionName?: string
  positionId?: string  // 用于跳转职位详情
  applicationId?: string  // 用于跳转投递记录
}
```

**SQL实现**:
```sql
-- 查询消息详情
SELECT 
  m.id,
  m.title,
  m.content,
  m.type,
  m.is_read AS "isRead",
  m.ref_id AS "refId",
  m.create_time AS "createTime",
  c.name AS "companyName",
  p.position_name AS "positionName",
  p.id AS "positionId",
  ja.id AS "applicationId"
FROM sys_message m
LEFT JOIN biz_job_application ja ON m.ref_id = ja.id
LEFT JOIN biz_position p ON ja.position_id = p.id
LEFT JOIN sys_company c ON p.company_id = c.id
WHERE m.id = #{id}
  AND m.receiver_id = #{studentId};

-- 标记为已读
UPDATE sys_message
SET is_read = 1
WHERE id = #{id}
  AND receiver_id = #{studentId}
  AND is_read = 0;
```

### 3. 获取未读消息数量

**接口**: `GET /api/mobile/messages/unread-count`

**响应数据**:
```typescript
interface UnreadCountResponse {
  total: number  // 总未读数
  systemCount: number  // 系统通知未读数
  interviewCount: number  // 面试通知未读数
  offerCount: number  // Offer通知未读数
}
```

**SQL实现**:
```sql
SELECT 
  COUNT(*) AS total,
  SUM(CASE WHEN type = 1 THEN 1 ELSE 0 END) AS "systemCount",
  SUM(CASE WHEN type = 2 THEN 1 ELSE 0 END) AS "interviewCount",
  SUM(CASE WHEN type = 3 THEN 1 ELSE 0 END) AS "offerCount"
FROM sys_message
WHERE receiver_id = #{studentId}
  AND is_read = 0
```

### 4. 标记消息为已读

**接口**: `PUT /api/mobile/messages/{id}/read`

**响应**: 成功/失败

**SQL实现**:
```sql
UPDATE sys_message
SET is_read = 1
WHERE id = #{id}
  AND receiver_id = #{studentId}
```

### 5. 批量标记已读

**接口**: `PUT /api/mobile/messages/read-all`

**请求参数**:
```typescript
interface ReadAllRequest {
  type?: number  // 可选：只标记某个类型的消息为已读
}
```

**SQL实现**:
```sql
UPDATE sys_message
SET is_read = 1
WHERE receiver_id = #{studentId}
  AND is_read = 0
  AND (#{type} IS NULL OR type = #{type})
```

### 6. 删除消息

**接口**: `DELETE /api/mobile/messages/{id}`

**SQL实现**:
```sql
DELETE FROM sys_message
WHERE id = #{id}
  AND receiver_id = #{studentId}
```

```typescript
// src/api/mobile/message.ts
import request from '@/utils/request'

export interface MessageQuery {
  pageNum: number
  pageSize: number
  type?: number
  isRead?: number
}

export interface MessageItem {
  id: string
  title: string
  content: string
  type: number
  isRead: number
  refId: string | null
  createTime: string
  companyName?: string
  positionName?: string
}

export interface MessageDetail extends MessageItem {
  positionId?: string
  applicationId?: string
}

export interface UnreadCountResponse {
  total: number
  systemCount: number
  interviewCount: number
  offerCount: number
}

// 获取消息列表
export const getMessages = (params: MessageQuery) => {
  return request<{ total: number; data: MessageItem[] }>({
    url: '/api/mobile/messages',
    method: 'get',
    params
  })
}

// 获取消息详情
export const getMessageDetail = (id: string) => {
  return request<MessageDetail>({
    url: `/api/mobile/messages/${id}`,
    method: 'get'
  })
}

// 获取未读消息数量
export const getUnreadCount = () => {
  return request<UnreadCountResponse>({
    url: '/api/mobile/messages/unread-count',
    method: 'get'
  })
}

// 标记消息为已读
export const markAsRead = (id: string) => {
  return request({
    url: `/api/mobile/messages/${id}/read`,
    method: 'put'
  })
}

// 批量标记已读
export const markAllAsRead = (type?: number) => {
  return request({
    url: '/api/mobile/messages/read-all',
    method: 'put',
    data: { type }
  })
}

// 删除消息
export const deleteMessage = (id: string) => {
  return request({
    url: `/api/mobile/messages/${id}`,
    method: 'delete'
  })
}
```

```typescript
// 获取消息类型颜色
export const getMessageTypeColor = (type: number): string => {
  const map: Record<number, string> = {
    1: '#1989fa',
    2: '#ff976a',
    3: '#07c160'
  }
  return map[type] || '#999'
}

// 格式化时间（相对时间）
export const formatRelativeTime = (time: string): string => {
  const now = new Date().getTime()
  const msgTime = new Date(time).getTime()
  const diff = now - msgTime

  const minute = 60 * 1000
  const hour = 60 * minute
  const day = 24 * hour

  if (diff < minute) {
    return '刚刚'
  } else if (diff < hour) {
    return `${Math.floor(diff / minute)}分钟前`
  } else if (diff < day) {
    return `${Math.floor(diff / hour)}小时前`
  } else if (diff < 7 * day) {
    return `${Math.floor(diff / day)}天前`
  } else {
    return time.split(' ')[0]  // 返回日期部分
  }
}
```

## 🔔 消息推送机制

### 企业端发送消息时机

1. **面试通知**：企业在"面试管理"页面安排面试时
2. **Offer通知**：企业在"面试管理"页面发放Offer时

### 消息创建逻辑（后端）

```java
// 企业安排面试时
public void scheduleInterview(InterviewDTO dto) {
    // 1. 保存面试信息
    Interview interview = saveInterview(dto);
    
    // 2. 创建面试通知消息
    MessageEntity message = new MessageEntity();
    message.setId(UUID.randomUUID().toString());
    message.setReceiverId(interview.getStudentId());
    message.setTitle("您投递的职位已安排面试");
    message.setContent(buildInterviewContent(interview));
    message.setType(2);  // 面试通知
    message.setIsRead(0);
    message.setRefId(interview.getApplicationId());
    message.setCreateTime(new Timestamp(System.currentTimeMillis()));
    
    messageMapper.insert(message);
}

// 企业发放Offer时
public void sendOffer(OfferDTO dto) {
    // 1. 保存Offer信息
    Offer offer = saveOffer(dto);
    
    // 2. 创建Offer通知消息
    MessageEntity message = new MessageEntity();
    message.setId(UUID.randomUUID().toString());
    message.setReceiverId(offer.getStudentId());
    message.setTitle("恭喜您获得Offer");
    message.setContent(buildOfferContent(offer));
    message.setType(3);  // Offer通知
    message.setIsRead(0);
    message.setRefId(offer.getApplicationId());
    message.setCreateTime(new Timestamp(System.currentTimeMillis()));
    
    messageMapper.insert(message);
}
```
