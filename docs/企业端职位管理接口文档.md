# 企业端职位管理接口文档

## 接口概述

企业端职位管理模块提供职位的发布、编辑、查询、上下架、删除等功能。

---

## 1. 获取职位列表（分页）

### 接口信息
- **接口路径**: `GET /api/company/job/list`
- **接口描述**: 分页查询企业发布的职位列表，支持按职位名称、审核状态、在招状态筛选
- **权限要求**: 企业HR角色

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | number | 是 | 页码，从1开始 |
| pageSize | number | 是 | 每页条数 |
| jobName | string | 否 | 职位名称（模糊查询） |
| audit | number | 否 | 审核状态：0=待审核, 1=已通过, 2=已驳回 |
| status | number | 否 | 在招状态：0=已下架, 1=在招 |

### 响应数据

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "data": [
      {
        "jobId": "job_123456",
        "jobName": "Java开发工程师",
        "salaryRange": "8-12K",
        "city": "成都",
        "education": "bachelor",
        "experience": "1-3",
        "tags": "五险一金,双休,年终奖",
        "audit": 1,
        "status": 1,
        "viewCount": 1523,
        "deliveryCount": 45,
        "createTime": "2026-02-01 10:00:00",
        "description": "负责公司核心业务系统的开发和维护...",
        "requirement": "1. 熟练掌握Java语言...",
        "contactPhone": "13800138000",
        "reason": null
      }
    ],
    "total": 50
  }
}
```

### SQL 实现示例

```sql
-- 基础查询
SELECT 
  j.job_id AS "jobId",
  j.job_name AS "jobName",
  j.salary_range AS "salaryRange",
  j.city,
  j.education,
  j.experience,
  j.tags,
  j.audit,
  j.status,
  j.view_count AS "viewCount",
  (SELECT COUNT(*) FROM biz_job_delivery d WHERE d.job_id = j.job_id) AS "deliveryCount",
  j.create_time AS "createTime",
  j.description,
  j.requirement,
  j.contact_phone AS "contactPhone",
  j.reason
FROM biz_job j
WHERE j.company_id = #{companyId}  -- 只查询自己公司的职位
  AND (#{jobName} IS NULL OR j.job_name LIKE CONCAT('%', #{jobName}, '%'))
  AND (#{audit} IS NULL OR j.audit = #{audit})
  AND (#{status} IS NULL OR j.status = #{status})
ORDER BY j.create_time DESC
LIMIT #{pageSize} OFFSET (#{pageNum} - 1) * #{pageSize};

-- 总数查询
SELECT COUNT(*) 
FROM biz_job j
WHERE j.company_id = #{companyId}
  AND (#{jobName} IS NULL OR j.job_name LIKE CONCAT('%', #{jobName}, '%'))
  AND (#{audit} IS NULL OR j.audit = #{audit})
  AND (#{status} IS NULL OR j.status = #{status});
```

### 业务规则

1. **权限控制**: 只能查询自己公司的职位（通过company_id过滤）
2. **投递数统计**: 通过子查询统计每个职位的投递数量
3. **排序规则**: 按创建时间降序排列

## 2. 发布新职位

### 接口信息
- **接口路径**: `POST /api/company/job/add`
- **接口描述**: 企业HR发布新职位，发布后状态为待审核（audit=0）
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "jobName": "Java开发工程师",
  "salaryRange": "8-12K",
  "city": "成都",
  "education": "bachelor",
  "experience": "1-3",
  "tags": "五险一金,双休,年终奖",
  "description": "负责公司核心业务系统的开发和维护...",
  "requirement": "1. 熟练掌握Java语言...",
  "contactPhone": "13800138000"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| jobName | string | 是 | 职位名称，2-100字符 |
| salaryRange | string | 是 | 薪资范围：3-5K/5-8K/8-12K/12-20K/20-30K/30K+/negotiable |
| city | string | 是 | 工作城市 |
| education | string | 是 | 学历要求：unlimited/college/bachelor/master/doctor |
| experience | string | 是 | 工作经验：unlimited/fresh/0-1/1-3/3-5/5-10/10+ |
| tags | string | 否 | 职位标签，逗号分隔 |
| description | string | 是 | 职位描述，至少10字符 |
| requirement | string | 是 | 任职要求，至少10字符 |
| contactPhone | string | 是 | 联系电话 |

### 响应数据

```json
{
  "code": 200,
  "msg": "发布成功，等待管理员审核",
  "data": {
    "jobId": "job_123456"
  }
}
```

### SQL 实现示例

```sql
INSERT INTO biz_job (
  job_id,
  company_id,
  job_name,
  salary_range,
  city,
  education,
  experience,
  tags,
  description,
  requirement,
  contact_phone,
  hr_id,
  status,
  audit,
  view_count,
  create_time
) VALUES (
  #{jobId},              -- 使用UUID生成
  #{companyId},          -- 从登录用户的HR信息获取
  #{jobName},
  #{salaryRange},
  #{city},
  #{education},
  #{experience},
  #{tags},
  #{description},
  #{requirement},
  #{contactPhone},
  #{hrId},               -- 从登录用户获取
  1,                     -- status: 1=在招
  0,                     -- audit: 0=待审核
  0,                     -- view_count: 初始为0
  CURRENT_TIMESTAMP
);
```

### 业务规则

1. **权限验证**: 
   - 只有企业HR可以发布职位
   - 从登录token中获取HR信息（hr_id, company_id）

2. **初始状态**:
   - `status = 1`（在招）
   - `audit = 0`（待审核）
   - `view_count = 0`

3. **字段验证**:
   - 职位名称：2-100字符
   - 职位描述和任职要求：至少10字符
   - 联系电话：手机号格式验证

## 2. 更新职位

### 接口信息
- **接口路径**: `PUT /api/company/job/update`
- **接口描述**: 更新职位信息，只能更新自己公司的职位
- **权限要求**: 企业HR角色

### 请求体

```json
{
  "jobId": "job_123456",
  "jobName": "高级Java开发工程师",
  "salaryRange": "12-20K",
  "city": "成都",
  "education": "bachelor",
  "experience": "3-5",
  "tags": "五险一金,双休,年终奖,股权激励",
  "description": "负责公司核心业务系统的开发和维护...",
  "requirement": "1. 熟练掌握Java语言...",
  "contactPhone": "13800138000"
}
```

### 响应数据

```json
{
  "code": 200,
  "msg": "更新成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE biz_job
SET 
  job_name = #{jobName},
  salary_range = #{salaryRange},
  city = #{city},
  education = #{education},
  experience = #{experience},
  tags = #{tags},
  description = #{description},
  requirement = #{requirement},
  contact_phone = #{contactPhone}
WHERE job_id = #{jobId}
  AND company_id = #{companyId}  -- 确保只能修改自己公司的职位
  AND hr_id = #{hrId};            -- 确保只能修改自己发布的职位
```

### 业务规则

1. **权限验证**:
   - 只能修改自己公司的职位
   - 只能修改自己发布的职位

2. **状态限制**:
   - 如果职位已被驳回（audit=2），修改后需要重新审核（audit=0）
   - 如果职位已下架（status=0），不允许修改

| jobId | string | 是 | 职位ID |

```json
    "companyLogo": "https://example.com/logo.png",
    "jobName": "Java开发工程师",
    "salaryRange": "8-12K",
    "city": "成都",
    "education": "bachelor",

    "requirement": "1. 熟练掌握Java语言...",

    "hrId": "hr_001",
    "hrName": "张三",
    "status": 1,
    "audit": 1,
    "reason": null,
    "viewCount": 1523,
    "createTime": "2026-02-01 10:00:00"
  }

```

### SQL 实现示例

```sql
SELECT 
  j.job_id AS "jobId",
  j.company_id AS "companyId",
  c.name AS "companyName",
  c.logo AS "companyLogo",
  j.job_name AS "jobName",
  j.salary_range AS "salaryRange",
  j.city,
  j.education,
  j.experience,
  j.tags,
  j.description,
  j.requirement,
  j.contact_phone AS "contactPhone",
  j.hr_id AS "hrId",
  h.name AS "hrName",
  j.status,
  j.audit,
  j.reason,
  j.view_count AS "viewCount",
  j.create_time AS "createTime"
FROM biz_job j
LEFT JOIN sys_company c ON j.company_id = c.id
LEFT JOIN biz_hr h ON j.hr_id = h.hr_id
WHERE j.job_id = #{jobId}
  AND j.company_id = #{companyId};  -- 确保只能查看自己公司的职位
```

## 4. 删除职位

### 接口信息
- **接口路径**: `DELETE /api/company/job/delete/{jobId}`
- **接口描述**: 删除职位（物理删除或软删除）
- **权限要求**: 企业HR角色

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| jobId | string | 是 | 职位ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "删除成功",
  "data": null
}
```

### SQL 实现示例

```sql
-- 物理删除
DELETE FROM biz_job
WHERE job_id = #{jobId}
  AND company_id = #{companyId}
  AND hr_id = #{hrId};

-- 或者软删除（推荐）
UPDATE biz_job
SET is_deleted = 1
WHERE job_id = #{jobId}
  AND company_id = #{companyId}
  AND hr_id = #{hrId};
```

### 业务规则

1. **权限验证**: 只能删除自己公司且自己发布的职位
2. **删除限制**: 
   - 如果有学生投递，建议提示HR是否确认删除
   - 建议使用软删除，保留历史数据
3. **关联数据**: 删除职位时需要考虑投递记录、面试记录等关联数据的处理

## 5. 下架职位

### 接口信息
- **接口路径**: `PUT /api/company/job/offline/{jobId}`
- **接口描述**: 下架职位，下架后学生无法看到此职位
- **权限要求**: 企业HR角色

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| jobId | string | 是 | 职位ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "下架成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE biz_job
SET 
  status = 0,
  update_time = CURRENT_TIMESTAMP
WHERE job_id = #{jobId}
  AND company_id = #{companyId}
  AND status = 1;  -- 只能下架在招的职位
```

### 业务规则

1. **状态变更**: status从1（在招）变为0（已下架）
2. **权限验证**: 只能下架自己公司的职位
3. **前端展示**: 下架后学生端不再显示此职位

## 6. 上架职位

### 接口信息
- **接口路径**: `PUT /api/company/job/online/{jobId}`
- **接口描述**: 上架职位，上架后学生可以看到此职位
- **权限要求**: 企业HR角色

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| jobId | string | 是 | 职位ID |

### 响应数据

```json
{
  "code": 200,
  "msg": "上架成功",
  "data": null
}
```

### SQL 实现示例

```sql
UPDATE biz_job
SET 
  status = 1,
  update_time = CURRENT_TIMESTAMP
WHERE job_id = #{jobId}
  AND company_id = #{companyId}
  AND audit = 1      -- 只能上架已通过审核的职位
  AND status = 0;    -- 只能上架已下架的职位
```

### 业务规则

1. **状态变更**: status从0（已下架）变为1（在招）
2. **审核限制**: 只有审核通过（audit=1）的职位才能上架
3. **权限验证**: 只能上架自己公司的职位

## 业务流程说明

### 职位发布流程

```
HR发布职位 → audit=0(待审核) → 管理员审核 → audit=1(通过) → 学生可见
                                    ↓
                              audit=2(驳回) → HR修改 → 重新审核
```

### 职位状态说明

**audit（审核状态）**：
- `0`: 待审核 - 刚发布或修改后等待管理员审核
- `1`: 已通过 - 管理员审核通过，学生可见
- `2`: 已驳回 - 管理员驳回，需要修改后重新提交

**status（在招状态）**：
- `1`: 在招 - 正常招聘中
- `0`: 已下架 - HR主动下架或管理员下架

### 权限控制

1. **HR身份验证**:
   ```sql
   -- 从登录token获取user_id，查询HR信息
   SELECT hr_id, company_id, name
   FROM biz_hr
   WHERE user_id = #{userId} AND status = 1;
   ```

2. **操作权限**:
   - 发布职位：需要HR角色
   - 修改职位：只能修改自己发布的职位
   - 查看职位：只能查看自己公司的职位

## 前端调用示例

```typescript
import { addJob, updateJob, getJobDetail } from '@/api/company'

// 1. 发布新职位
const publishJob = async () => {
  await addJob({
    jobName: 'Java开发工程师',
    salaryRange: '8-12K',
    city: '成都',
    education: 'bachelor',
    experience: '1-3',
    tags: '五险一金,双休,年终奖',

// 2. 编辑职位
const editJob = async (jobId: string) => {
  // 先获取详情
  const res = await getJobDetail(jobId)
  const job = res.data
  
  // 修改后提交
  await updateJob({
    jobId: job.jobId,
    jobName: job.jobName,
    // ... 其他字段
  })
}
```

## 注意事项

1. **HR信息获取**: 
   - 从登录token中获取user_id
   - 通过biz_hr表查询hr_id和company_id
   - 确保HR状态为正常（status=1）

2. **数据验证**:
   - 职位名称、描述、要求不能为空
   - 联系电话格式验证
   - 薪资范围、学历、经验使用固定枚举值

3. **审核机制**:
   - 新发布的职位需要管理员审核
   - 修改已通过的职位可能需要重新审核（根据业务需求）
   - 被驳回的职位修改后自动变为待审核状态

4. **字段说明**:
   - `tags`: 逗号分隔的字符串，前端需要split处理
   - `education`/`experience`: 使用字典值，前端需要映射为中文显示
   - `audit`: 0=待审核, 1=已通过, 2=已驳回
   - `status`: 0=已下架, 1=在招
